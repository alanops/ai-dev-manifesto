# ✅ Pull Request Checklist (Optional)

Use this to keep your PRs clean, clear, and AI-aware:

## 🧪 Functionality & Testing
- [ ] I tested this locally and verified key behavior works
- [ ] Edge cases are handled (or documented as known limitations)
- [ ] No breaking changes without team discussion

## 🧠 Code Understanding & Quality
- [ ] I understand every line — even if AI helped write it
- [ ] Complex logic is commented or documented
- [ ] Code follows existing patterns and conventions
- [ ] No sensitive data (keys, tokens, passwords) in the code

## 📋 PR Structure & Communication
- [ ] This PR is focused and solves one clear problem
- [ ] PR description explains the "why" not just the "what"
- [ ] Commit messages are meaningful and specific
- [ ] Large files or generated code are explained

## 🤖 AI-Assisted Work (If Applicable)
- [ ] AI-generated sections are reviewed and understood
- [ ] I can explain any AI suggestions I accepted
- [ ] Generated code fits our coding standards
- [ ] No copy-paste without understanding

## 🔍 Review Readiness
- [ ] PR is reasonably sized (< 500 lines unless unavoidable)
- [ ] Comments and docs are minimal but meaningful
- [ ] I've self-reviewed the diff before requesting review
- [ ] Related documentation is updated if needed
