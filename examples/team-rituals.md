# 🔄 Team Rituals for AI-Era Development

Ideas for fostering quality, clarity, and trust in AI-assisted workflows.

## 🎯 Daily/Weekly Practices

### "Ask Early" Culture
- **Standup Addition**: "Anyone stuck or unsure about something?"
- **Slack Channel**: `#stuck-or-unsure` for quick help requests
- **Ritual**: No shame in raising flags — celebrate asking over struggling silently

### AI Transparency
- **"AI Assisted" PR Labels**: Tag PRs that used significant AI help
- **Show Your Work**: Brief comment on AI-generated complex logic
- **Share Wins**: Weekly Slack thread for cool/clever/surprising AI solutions

## 🧪 Code Review Enhancements

### Review with Context
- **Question-First Reviews**: Start with "What problem does this solve?" 
- **Pairing Sessions**: Reviewer and author walk through AI-generated code together
- **Learning Moments**: Use reviews to share AI techniques that worked well

### Quality Gates
- **AI Code Callouts**: `// AI-generated, reviewed by [name]` for transparency
- **Understanding Test**: "Can you explain this logic?" during code reviews
- **Documentation Triggers**: Complex AI solutions get a brief explanation

## 📚 Team Learning

### Retro Prompts
- "What did AI accelerate or confuse this week?"
- "Where did we catch AI mistakes before they became problems?"
- "What AI workflows are making us more effective?"

### Knowledge Sharing
- **AI Tool Show-and-Tell**: Monthly demo of useful AI workflows
- **Pattern Library**: Document effective AI prompts or approaches
- **Learning Logs**: Optional but encouraged - share what AI taught you

## ⚖️ Balancing Speed & Quality

### Pair Programming Variations
- **AI + Human Pairing**: One person uses AI, one doesn't — compare approaches
- **Review Pairing**: Pair review sessions for AI-heavy PRs
- **Teaching Moments**: Junior dev with AI + senior dev without AI

### Delivery Rhythms
- **Sprint Hygiene**: Regular check-ins on code quality vs delivery pressure
- **Tech Debt Tracking**: Flag when AI shortcuts create future maintenance burden
- **Quality Metrics**: Track review time, bug rates, etc. to measure AI impact

## 🛠 Optional Automation

### PR Templates
```markdown
## Type of Change
- [ ] New feature
- [ ] Bug fix  
- [ ] Refactoring
- [ ] AI-assisted development

## AI Usage (if applicable)
- [ ] I reviewed all AI-generated code
- [ ] Complex AI logic is documented
- [ ] I can explain any AI suggestions I accepted
```

### Slack Integrations
- **AI Wins Bot**: Share cool AI solutions automatically
- **Code Quality Reminders**: Gentle nudges about review best practices
- **Stuck Helper**: Easy way to request help when blocked

*Remember: Not every team needs every ritual.*
